聊天气泡排版优化方案

请选择你倾向的优化方向

📐 分析发现的当前问题

中文排版

  • 字体栈是英文优先，中文渲染不佳
  • 行高 1.5 对中文偏紧
  • 段落间距为 0，文字全挤在一起
  • 字间距对中文阅读不友好

层次与布局

  • 标题/列表/代码/段落间无间距
  • markdown-styles.css 与 index.css 样式冲突
  • 缺少垂直节奏感（呼吸感）

交互细节

  • 操作按钮 hover 才可见（移动端不可用）
  • 出现动画较简单
  • 打字指示器可更精致

方案 A：轻量优化（推荐）

最小的改动带来最大收益

  • 字体栈：加入 PingFang SC / Noto Sans SC 中文优先顺序
  • 行高：正文 1.7 → 更适合中英文混排
  • 段间距：段落间距 0.6em，标题上下间距 0.4em
  • 气泡间距：调整为 20px（原 16px），内边距微调
  • 冲突修复：统一两套 CSS 的间距体系
  • 操作按钮：轻微透明度，而非完全隐藏

方案 B：中重度重新设计

更全面的排版系统 + 交互升级

  • 包含方案 A 所有改动
  • 设计 Token 化：统一间距/字号/圆角变量体系
  • 消息分组：连续同角色消息合并，去掉多余头像
  • 代码块升级：行号 + 更好的语法高亮主题
  • 过渡动效：消息入场的阶梯动画，打字指示器的波形动画
  • 移动端适配：操作按钮常驻，气泡宽度 95%
  • 可读性优化：较长的 AI 回复自动添加阅读进度指示

A：轻量优化

最小改动，中文字体 + 间距体系 + 样式统一

B：全面升级

排版系统化 + 交互升级 + 消息分组 + 动效
